### Why these three?

Action 1 is the highest-leverage fix in the repo: the treemap is `/assess`'s headline artefact, and it **crashes on the current default Python** (`requires-python = ">=3.12"` resolves to 3.14.4 via `uv`, and 3.14's argparse rejects the unescaped `%`). CI never caught it because the test suite imports the modules but never exercises the `argparse` path — so a one-line escape plus a `--help` smoke test closes a real gap. Actions 2 and 3 are the two false positives this very run produced about the toolkit itself: a self-referential liveness rung and a lying-map flag on a doc edited the same day. Fixing them makes the tool's own output trustworthy on the exact repos most likely to run it (repos that document tooling).
